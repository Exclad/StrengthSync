## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual CSS with design tokens) |
| Preset | not applicable |
| Component library | none (plain React 18 via CDN + Babel standalone) |
| Icon library | Custom inline SVG icons in `static/src/icons.jsx` — `IconSettings` already exists |
| Font | Inter Tight (400/500/600/700/800) + JetBrains Mono (400/500/600) via Google Fonts |

**No shadcn gate applicable.** This project uses a hand-rolled CSS token system defined in `templates/index.html` `:root` and `[data-theme="dark"]` blocks. All new UI must use these tokens — no new tokens are needed for Phase 6.

**React pattern:** CDN React 18 + Babel standalone transpiles JSX in-browser. No build step. New screens are added as `<script type="text/babel" src="...">` tags in `templates/index.html`. Components are assigned to `window.ComponentName` at end of each file.

|------|-------------|------------|-------|

| Good | `--good: #2F9E5D` (light) / `#5DD496` (dark) | same | Mapped state, success confirmation |

Accent (`--accent`) is reserved for: active nav-pill tab, focus outlines, the accent-stripe overlay on the brand mark, and `.chip.accent` status chips. Accent is NOT used for primary action buttons — those use `--ink` (`.btn-dark`).

**Dark mode brand mark fix (D-06):**
The `.brand-mark` currently uses `background: var(--ink)` and `color: var(--accent)`. In light mode this renders dark-background + lime letter — readable. In dark mode `--ink` becomes `#F4F3EE` (near-white), making the mark nearly invisible against the `--bg: #0C0C0A` topbar.

Fix specification: add `[data-theme="dark"] .brand-mark` override in `templates/index.html`:
```css
[data-theme="dark"] .brand-mark {
  background: var(--surface-2);   /* #1E1E1B — visible dark surface */
  border: 1px solid var(--line);  /* subtle edge definition */
}
```
The `color: var(--accent)` "S" letter remains visible because `--accent` is `#E6FF3D` in both themes. The `::before` stripe gradient uses `var(--accent)` and also remains visible. This is a pure CSS addition — no JSX change needed.

### 1. Settings Screen (`static/src/screen_settings.jsx`)

Structure mirrors Library and History screens: eyebrow + 32px h-display heading, then cards.

**Nav integration:** Add `["settings","Settings"]` to the nav array in `shell.jsx` line 20. Add `{page === "settings" && <ScreenSettings/>}` in `app.jsx` alongside Library and History.

Settings screen layout: single-column stack of `.card` elements, each containing one settings group. Card padding: 24px (`card-pad`). Gap between cards: 16px.

**Settings groups and their form elements:**

#### Group A: Timezone Default
- Label: eyebrow style — `TIMEZONE DEFAULT` — `var(--ink-3)`, JetBrains Mono 11px, letter-spacing 0.12em
- Description: 13px `var(--ink-3)` — "Pre-fills the timezone picker on the Upload screen. Auto-detected from your browser when not set."
- Control: same filter input + `<select size={5}>` pattern as Upload screen's timezone picker
- Persistence: `localStorage.setItem('ss-timezone', value)` on change
- Auto-detect button: `.btn .btn-ghost .btn-sm` — label "Use browser timezone" — calls `Intl.DateTimeFormat().resolvedOptions().timeZone` and sets the value
- Confirmed state: shows `<IconCheck size={12}/>` + timezone string in `var(--good)`, JetBrains Mono 12px (matches Upload screen confirmation pattern)

#### Group B: Export Filename Pattern
- Label: `EXPORT FILENAME` — same eyebrow style
- Description: 13px `var(--ink-3)` — "Customize the output filename. Use {date} for workout date and {workout} for workout name."
- Control: `<input type="text">` styled as: `border: 1px solid var(--line); border-radius: 8px; padding: 8px 10px; background: var(--surface); color: var(--ink); font-size: 13px; width: 100%`
- Default value: `{date}_{workout}.fit` (pre-populated from localStorage or this default)
- Live preview: rendered below the input in a `var(--surface-2)` inset box, 12px padding, border-radius 8px, border `1px solid var(--line)`. Preview label: `PREVIEW` eyebrow in JetBrains Mono 10px. Preview value: JetBrains Mono 13px weight 600 `var(--ink)`.
- Persistence: `localStorage.setItem('ss-filename-pattern', value)` on change (debounce 300ms)

#### Group C: Output Folder
- Label: `OUTPUT FOLDER` — eyebrow style
- Description: 13px `var(--ink-3)` — "Path where merged FIT files are saved. Defaults to the output/ folder inside the app directory."
- Control: `<input type="text">` — same styling as filename input above
- Current value badge: `.chip.neutral.mono` showing current resolved path (fetched from `GET /api/settings` or hardcoded default `output/`)
- Implementation decision (Claude discretion): use plain text input, not OS file picker. OS file picker requires `webkitdirectory` attribute or a server-side endpoint. Text input is sufficient for a developer-grade local tool. Users can paste the full path.
- Persistence: `localStorage.setItem('ss-output-folder', value)` on blur

#### Group D: Database Reset
- Label: `DANGER ZONE` — eyebrow style, color `var(--bad)` instead of `var(--ink-3)`
- Description: 13px `var(--ink-3)` — "Permanently deletes all saved exercise mappings. Future syncs will need to re-map exercises from scratch."
- Button: `.btn` with inline style `background: color-mix(in oklab, var(--bad) 15%, var(--surface)); color: var(--bad); border: 1px solid color-mix(in oklab, var(--bad) 30%, var(--line)); border-radius: 10px;` — label "Clear all exercise mappings"
- Confirmation modal: same pattern as Library screen's delete confirmation modal — fixed overlay `rgba(0,0,0,0.4)`, `.card` centered, max-width 380px, padding 28px. Contains:
  - Heading 16px weight 700: "Clear all exercise mappings?"
  - Body 13px `var(--ink-3)`: "This will permanently delete all {N} saved mappings. Future syncs will re-map exercises from scratch. This cannot be undone."
  - Button row: `Cancel` (`.btn.btn-ghost`) + `Clear all mappings` (`.btn` with `background: var(--bad); color: #fff; border: none`)
- API call: `POST /api/map/reset` — on success, show `.chip.good` inline: "All mappings cleared."

### 2. Timezone Auto-Detect (Upload Screen modification)

**Where it runs:** On `ScreenUpload` mount (inside the existing `useEffect` that fetches timezones), after the timezone list loads.

**Logic:**
```
1. Check localStorage.getItem('ss-timezone') — if set, use it and skip auto-detect
2. If not set, call Intl.DateTimeFormat().resolvedOptions().timeZone
3. Verify the detected timezone exists in the fetched timezones list
4. If found, set as the selected timezone and show auto-detect indicator
5. If not found, leave picker unselected (existing behavior)
```

**Auto-detect indicator:** A small note below the timezone select, 12px `var(--ink-3)`:
- When auto-detected: "Auto-detected from your browser · <button class='btn btn-ghost btn-sm'>Change in Settings</button>"
- When from saved preference: "Saved preference · <button class='btn btn-ghost btn-sm'>Change in Settings</button>" — button calls `setPage("settings")` (requires passing `setPage` as a prop or using a shared callback)

**Post-sync persistence (D-04):** In `app.jsx` `handleRestart` is NOT the trigger. Persistence happens when the user advances from Upload to Match (i.e., in `ScreenUpload`'s `handleContinue` success path): `localStorage.setItem('ss-timezone', timezone)`.

### 4. Error UX Polish

**General error banner pattern (confirmed existing):**
```jsx
<div className="chip bad" style={{
  display: 'flex', alignItems: 'center', gap: 8,
  padding: '10px 14px', borderRadius: 10, marginTop: 12
}}>
  <IconWarn size={14}/>
  <span style={{ fontSize: 13 }}>{errorMessage}</span>
</div>
```
This pattern is used in `screen_upload.jsx` and `screen_map.jsx`. It is correct and must NOT be restyled — only the copy inside it changes.

**Error copy templates (all errors must follow: problem + cause + action):**

| Screen | Trigger | Current copy | Required copy |
|--------|---------|-------------|---------------|
| Upload | Non-FIT file uploaded | "Upload failed." (generic) | "That file doesn't look like a Garmin FIT file. Export from Garmin Connect → Activity → ⋯ → Export original." |
| Upload | Non-Hevy CSV uploaded | "Upload failed." (generic) | "The CSV doesn't match Hevy's export format. In Hevy: Profile → Settings → Export Workout Data → Download." |
| Upload | Network error | "Network error. Is the app running?" | Keep as-is — already actionable. |
| Upload | FIT parse error | Backend message passed through | Prefix with "Couldn't read the FIT file. " and append " Try exporting a fresh copy from Garmin Connect." |
| Match | No matching workout found | Backend message | "No Hevy workout found within 30 minutes of this Garmin session. Check your timezone is set correctly, or use manual match." |
| Match | Multiple ambiguous matches | Backend message | "Found {N} possible matches. Pick the correct Hevy workout below." |
| Map | Confirm failed | "Confirm failed." | "Couldn't save that mapping. Check the app is still running and try again." |
| Map | Network error | "Network error." | "Network error — couldn't reach the app. Refresh and try again if the problem persists." |
| Export | FIT generation failed | Backend message | Prefix with "Export failed. " and append " Your original files are still available — try re-uploading." |
| Settings | DB reset failed | (new) | "Couldn't clear mappings. Check the app is still running and try again." |

**Exercise mapping UNRESOLVED affordance improvement (D-10):**

The current "Search all {N} Garmin exercises" button is a small `.btn.btn-ghost.btn-sm` in the top-right of the suggestions section. For exercises with status `unmapped` (no suggestions at all) or `needs-review` (low confidence), the path to resolution is unclear.

Required changes to `MappingDetail` in `screen_map.jsx`:

1. When `exercise.status === 'unmapped'` (no suggestions): replace the sparse "AI suggestions" header with a prominent call-to-action:
   - Yellow/warn-tinted inset box: `background: color-mix(in oklab, var(--warn) 8%, var(--surface)); border: 1px solid color-mix(in oklab, var(--warn) 25%, var(--line)); border-radius: 10px; padding: 16px 20px`
   - Row with `<IconWarn size={14}/>` color `var(--warn)` + text 13px weight 600: "No automatic match found"
   - Sub-text 12px `var(--ink-3)`: "Search for a Garmin exercise name below, or skip to use Garmin's original data."
   - Immediately below: the search input is **already expanded** (`showSearch` forced to `true` when status is `unmapped`)
   - Skip button is promoted to `.btn.btn-ghost` (not `.btn-sm`) with clearer label: "Skip — keep Garmin's original exercise"

2. When `exercise.status === 'needs-review'` (low-confidence suggestions exist): add a note above suggestion list — 12px `var(--warn)` italic — "Low confidence match. Review before continuing."

3. The `{unresolvedCount} UNRESOLVED` chip in the bottom action row (line 289) is upgraded: when `unresolvedCount > 0`, show it as a `.chip.bad` (not `.chip.warn`) to increase urgency. Change copy to `{unresolvedCount} NEED ACTION`.
