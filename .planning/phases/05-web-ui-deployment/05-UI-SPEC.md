### Accent Usage Contract

`--accent` (electric lime) is reserved for:
1. Primary button background (`.btn-primary`)
2. Active step progress bar fill (`.rail-step.active .bar::after`)
3. Focus ring (`outline: 2px solid var(--accent)`)
4. Connector SVG circle fill (matched workout indicator)
5. Mapping arrow circle fill (when exercise is mapped)
6. Export-complete success icon background (ScreenDone)
7. Brand mark foreground text color

`--accent` is NOT used for: text on white/surface backgrounds, borders alone, hover states on secondary elements.

`--accent-2` (burn orange) is reserved for:
1. HR chart stroke and area gradient
2. Drag-drop zone hover border
3. Progress ring stroke in ScreenMap
4. Selected list item left border in exercise list
5. "Replaced by Hevy" audit icon color

Destructive color (`--bad` / `#D84848` light, `#FF6B6B` dark) is reserved for:
1. Error banners (inline, not modal)
2. NO MATCH chip background tint
3. Max HR threshold line in HR chart
4. `.chip.bad` status elements only

## Screen Interaction Contract

### Screen 1 ‚Äî Upload (UI-01)

**Entry state:** Empty ‚Äî no files, no Hevy mode selected.

The drag-drop zone (`2px dashed var(--line-2)` border on a `var(--surface-2)` background) is the primary visual anchor of this screen. It occupies the dominant left column and drives the first user action.

**Gates:**
- "Continue to matching" button: `opacity: 0.4; cursor: not-allowed` until `fitFiles.length > 0 AND hevyMode` is set.
- Only file mode (`hevyMode: "file"`) is implemented for Phase 5. OAuth option (`hevyMode: "api"`) renders as deferred ‚Äî show option but disable or show "Coming soon" on click (per deferred list).

**Drag-and-drop zone:**
- Border: `2px dashed var(--line-2)` idle ‚Üí `2px dashed var(--accent-2)` on drag-over.
- Background: `var(--surface-2)` idle ‚Üí `color-mix(in oklab, var(--accent-2) 6%, var(--surface))` on drag-over.
- Accepts: `.fit` files only (multiple OK).

**Timezone picker (D-12):**
- Rendered below the file upload panels, above the CTA row.
- Implemented as a `<select>` with search-filter (type to filter visible options).
- Populated from `GET /api/timezones` on mount.
- Common zones floated to top: `America/New_York`, `America/Los_Angeles`, `America/Chicago`, `Europe/London`, `Europe/Berlin`, `Asia/Tokyo`, `Asia/Singapore`, `Australia/Sydney`.
- No free-text input ‚Äî invalid values must be blocked at the select level.
- Required before "Continue" gate opens (along with files and Hevy mode).

**"Use demo data" button:** Visible and functional in dev. Labeled `Use demo data` with `IconZap`. Loads hardcoded seed files.

### Screen 2 ‚Äî Match (UI-01, MATCH-02, MATCH-03)

**Layout:** Three-column grid ‚Äî `1fr auto 1fr` ‚Äî Garmin cards | Connector SVGs | Hevy pickers.

**Connector states:**
- Matched: accent-filled circle with checkmark, `var(--ink)` dashed line, confidence % label.
- Unmatched: warn-bordered circle with warn exclamation, `var(--line-2)` dashed line, "NO MATCH" label.

**Manual override:** Unmatched Hevy picker shows a list of all Hevy workouts as buttons; "Skip this workout" button at bottom.

**No export gate on this screen** ‚Äî all Garmin workouts can proceed (unmatched ones are skipped).

### Screen 3 ‚Äî Map exercises (UI-02, MAP-03) ‚Äî HERO SCREEN

**Layout:** `360px left sticky list | 1fr right detail pane`.

**Export gate:** "Preview merge" button disabled (`opacity: 0.4`) until all exercises are either `status: "mapped"` or explicitly skipped. Chip `warn` "N UNRESOLVED" shown while gate is blocked.

**Exercise statuses:**
- `mapped` (confidence ‚â• 0.70 auto or user confirmed): green dot, chip.good "MAPPED ¬∑ N%"
- `needs-review` (confidence < 0.70): amber dot, chip.warn "LOW CONFIDENCE ¬∑ N%"
- `unmapped` (no suggestion): red dot, chip.bad "NO MATCH"
- `cardio` (from `skipped_cardio`): neutral dot, chip.neutral "CARDIO ‚Äî SKIPPED" ‚Äî no user action required, does not block export

**Suggestion interaction:** Top suggestion highlighted with `var(--surface-2)` background + `‚Üµ` kbd hint. Click any suggestion to accept. "Search all N Garmin exercises" button opens an inline searchable list.

**Skip action:** "Skip this exercise ‚Äî keep Garmin's guess" button at bottom of suggestions panel. Records `skip` status ‚Äî clears the export block for this exercise.

**Auto-advance:** After accepting a mapping, focus advances to the next UNRESOLVED exercise in the list.

**Mapping persistence:** Each confirmed mapping POSTs to `POST /api/map/confirm`. On subsequent loads, pre-resolved exercises show as `mapped` with `confidence: 1.0`.

### Screen 4 ‚Äî Preview (MERGE-04)

**KPI strip:** 6-column grid. `preserved` KPIs show green dot; `replaced` KPIs show burn-orange dot. All values sourced from `build_preview()` via `POST /api/preview`.

**HR chart:** SVG, `viewBox="0 0 1140 260"`, responsive width. If `FitWorkout.heart_rate_samples` is empty, replace chart with an empty-state card: "No HR data ‚Äî this workout was recorded without a heart rate sensor." Use existing card structure with `IconInfo` and `var(--ink-3)` text.

**Before/Merged/Diff tab toggle:** `.nav-pill` with three buttons. Active tab: `background: var(--ink); color: var(--bg)`.

**CTA:** `.btn.btn-primary.btn-lg` ‚Äî "Generate & export .fit" with `IconDownload`. Only button that uses `btn-primary` (lime) in the app flow.

### Screen 5 ‚Äî Export/Done (UI-03)

**Generating phase:** Full-screen centered spinner (`screen_done.jsx` SVG animation with `animateTransform`). Spinner shows while `POST /api/export` is in flight (D-17 ‚Äî no artificial delay). Progress log lines shown below spinner using `IconCheck + mono` list.

**Ready phase:** Triggered on API response. Shows download card with file name (`{YYYY-MM-DD}-STRENGTH-merged.fit`), size, FIT Protocol version, and CRC-8 checksum (`.mono`). "Download .fit file" button: `.btn.btn-primary.btn-xl` with `IconDownload`.

**Validation banner:** Green left-border card (`borderLeft: "3px solid var(--good)"`) with success message when CRC passes.

**Restart:** `.btn.btn-ghost` "Sync another workout" ‚Äî resets React state and returns to step 0.
